# Truthwatcher v0.1.0 Known Limitations

Truthwatcher v0.1.0 is a local, fixture-backed evidence-to-understanding proof of concept. It is designed to show that raw evidence can be preserved, parsed, modeled, and inspected with provenance intact.

## Included in v0.1.0

- A single local Go binary with embedded migrations and embedded UI assets.
- PostgreSQL-backed local state.
- Fake fixture-backed discovery using included Junos and IOS-XR sample outputs.
- Raw evidence persistence with hashes and metadata.
- Parser persistence from stored discovery evidence into assets, facts, relationships, parse results, and identity candidates.
- Read APIs for discovery runs, evidence, assets, facts, asset-scoped relationships, graph neighborhoods, identity candidates, discovery plans, and architecture seeds.
- Embedded UI routes for the dashboard, discovery runs, assets, graph, discovery plans, architecture seeds, deterministic questions, and about/help.
- Local seed SQL for demo architecture context.

## Explicitly excluded from v0.1.0

- Production discovery collectors.
- Network write operations, configuration changes, remediation, commits, reloads, clears, or arbitrary command execution.
- Credential handling, admin features, organizations, RBAC, or production auth.
- External LLM calls, hosted services, external integrations, containers, microservices, or network write capabilities.
- Continuous scheduling, production source-of-truth synchronization, or live IPAM/EMS/Nautobot/NetBox adapters.

## Evidence and seeded context boundaries

Seeded architecture context is operator-provided planning context. It is not observed proof, and it must not be interpreted as device evidence.

Raw evidence remains the source of truth. Assets, facts, relationships, graph views, identity candidates, and deterministic answers are derived views that must remain explainable by evidence references or clearly labeled as seeded, inferred, conflicting, unknown, or awaiting review.
