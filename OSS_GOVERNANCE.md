# Open Source Governance

This project is governed as a maintainer-led open source project.

## Governance Model

The project follows a maintainer-led governance model. Maintainers guide the direction of the project, review contributions, and make final decisions on changes.

## Contributions

Pull requests are welcome. Contributors are encouraged to open issues or pull requests for bug fixes, improvements, documentation updates, and proposed features.

## Merge Policy

Project owners are responsible for merging pull requests. Pull requests should be reviewed for alignment with project goals, quality, and maintainability before merge.

## Roadmap

The project roadmap is tracked in Git. Roadmap updates should be committed to the repository so planning decisions remain visible and versioned.

## Architecture Decision Records

Significant changes should be ADR-driven. Architecture Decision Records should document the context, decision, and consequences of major technical or governance changes.
