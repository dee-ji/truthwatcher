Rules:
- Keep Truthwatcher as a single Go binary plus PostgreSQL unless explicitly asked otherwise.
- Do not introduce Docker, Kubernetes, microservices, message brokers, or cloud dependencies.
- Do not introduce write-capable network automation.
- Do not build observability features.
- Do not build a full chat platform before the evidence kernel exists.
- Prefer boring, explicit Go code.
- Add tests where practical.
- Update ROADMAP.md only with completed work and next steps.
- Do not rewrite vision documents unless directly asked.


Task: implement the stable knowledge model: Asset, Fact, Relationship.

Deliverables:
1. Migrations for:
   - `assets`
   - `facts`
   - `relationships`
2. Assets should support:
   - id
   - type
   - identity_key
   - vendor nullable
   - model nullable
   - serial nullable
   - system_mac nullable
   - metadata JSONB
3. Facts should support:
   - asset_id
   - name
   - value JSONB
   - source
   - confidence
   - evidence_id
4. Relationships should support:
   - source_asset_id
   - target_asset_id
   - relationship_type
   - confidence
   - evidence_id
   - metadata JSONB
5. Add repositories for basic create/get/list.
6. Add uniqueness rules carefully:
   - do not assume hostname or IP is unique identity
   - identity_key should be generated by system logic
7. Document identity strategy in `steering-docs/DATA_MODEL.md` if not already present.

Constraints:
- Do not over-model every network concept yet.
- Do not create vendor-specific tables.
- Use JSONB for unknown surface area.
